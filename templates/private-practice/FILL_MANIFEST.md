# FILL_MANIFEST — Solo Private-Practice Template

This is the **fill engine** for the template. An AI (or a person) makes one customer's site by:

1. Reading the customer's facts from a `practice.md` (or equivalent intake).
2. Replacing every `{{token}}` below with the matching field (string substitution across `index.html` + `app.js`).
3. Resolving every `<!-- AI-GENERATE:* -->` block by running its prompt and pasting the result in place of the neutral placeholder that follows the comment.
4. Replacing the two placeholder assets (`assets/placeholder-headshot.svg`, `assets/placeholder-logo.svg`) if the customer supplies a headshot / logo — see **Assets to upload**.

**Architecture note (important):** This is a JavaScript single-page app. `index.html` is only the nav + footer shell; **all page body content is generated by `app.js`** (route templates + the `posts` data array). Tokens and AI-GENERATE blocks therefore live in BOTH `index.html` and `app.js` — substitution must run over both files. `styles.css` and `tweaks.js` are pure design/dev and contain no customer content.

**Counts:** 53 unique tokens · 37 AI-GENERATE blocks · 1 HTML file (`index.html`) + 1 content JS file (`app.js`).

**Tone contract (applies to every AI-GENERATE block):** peer-to-peer clinician voice — plain, grounded, specific, warm. No marketing fluff, no hype, no clichés ("unlock your best self", "journey to wholeness"). Never invent credentials, outcomes statistics, license numbers, or modalities the clinician isn't actually trained in. When a field is unknown, prefer omission over fabrication. US crisis numbers 988 / 911 are hard-coded and correct for US practices; localize for non-US.

### Single-state / no-office / no-method practices
- **Single-state telehealth-only:** set `{{location}}` to the clinician's base city, drop `{{office_detail}}`/`{{address_*}}` to "" (and remove the office rows/cards), set `{{license_2_*}}` and `{{credential_line_2/3}}` to "".
- **No physical office:** blank the office quick-fact + the contact "Office" block; lead with telehealth.
- **No signature method / no program:** see the `method_*` and `journey_*` AI-GENERATE rows — each has a documented fallback that turns the section into a generic "how I work" / "arc of therapy" treatment. You may also hide the `method`/`journey` nav links + footer links if the practice genuinely has neither.

### Notes on JS-array blocks
Several blocks (`method_intro_cards`, `populations_chips`, `career_timeline`, `modalities`, `method_steps`, `journey_phases`, `fees_bullets`, `fees_faq`, `writing_tags`, `blog_posts`, `state_options`) replace a **JavaScript array/markup literal**, not prose. The fill engine must emit syntactically valid JS (balanced brackets, escaped apostrophes `\'`, HTML-escaped `&` where the string lands in HTML). After filling, a quick `node --check app.js` catches any broken literal.

## 4. FILL CHECKLIST (order of operations)

1. Gather `practice.md`. Fill every token in §1 (run substitution over **both** `index.html` and `app.js`).
2. Resolve every AI-GENERATE block in §2 (replace the placeholder that follows each comment; you may leave or delete the comment).
3. Upload real headshot/logo if available; update the `url()` extension if needed (§3).
4. Set `{{current_year}}`; handle single-state / no-office / no-method cases (§1 sub-notes).
5. Localize crisis lines for non-US practices (988/911 are US-only).
6. Validate: `node --check app.js`, then serve the folder and load `index.html` — confirm the home + about routes paint with no leftover `{{tokens}}` and no console errors.
7. Have counsel review the Privacy / Terms / Accessibility copy before publishing.
